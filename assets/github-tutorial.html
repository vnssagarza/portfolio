    Background

    GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere. This tutorial teaches you GitHub essentials like repositories, branches, commits, and pull requests. Learn more about the terminology used in this tutorial in the glossary.

    Before you start

    Before you start this tutorial, you need a GitHub account and Internet access. This tutorial is intended for new users. You don‚Äôt need to know how to code, use the command line, or install Git (the version control software that GitHub is built on).

    Create a repository

    1. In the upper-right corner of any page, go to the + drop-down menu, and select New repository.

    2. In the Repository name box, enter hello-world.

    A repository is usually used to organize a single project. Repositories can contain folders and files, images, videos, spreadsheets, and data sets ‚Äì anything your project needs.

    1. In the Description box, enter a short description.

    2. Select whether your repository will be Public or Private.

    3. Select Add a README file.

    Repositories often include a README file that provides information about your project.

    1. Select Create repository.

    Create a branch

    1. Select the Code tab of your hello-world repository.

    2. Above the file list, go to the dropdown menu and select main.

    3. Enter a branch name, readme-edits, into the text box.

    Note: A branch is a copy, or snapshot, of the default main branch.

    1. Select Create branch: readme-edits from main.

    Now you have two branches, main and readme-edits.

    Make and commit changes

    1. In the readme-edits branch, select the README.md file.

    2. To edit the file, select edit .

    3. In the editor, enter information about yourself.

    üí° Tip: README files are written in the plain text Markdown language. You can use this cheat sheet to get started with Markdown syntax.

    1. Select Commit changes‚Ä¶

    Note: These changes will be made only to the README file on your readme-edits branch.

    1. In the Propose changes box, enter a commit message that describes your changes.

    Note: Commit messages capture the history of your changes so that other contributors can understand what you‚Äôve done and why.

    1. Select Propose changes.

    Open a pull request

    1. Select the Pull requests tab of your repository.

    Note: Pull requests are requests for someone to review and pull in your changes and merge them into their branch.

    1. Select New pull request.

    2. In the Example Comparisons box, select the branch you made, readme-edits, to compare with main.

    3. Before submitting your changes, review the diffs on the Compare page.

    Note: The changes, additions, and subtractions are shown in different colors.

    1. Select Create pull request.

    2. Enter a title for your pull request and a brief description of your changes.

    üí° Tip: You can include emojis and drag and drop images and gifs.

    1. Optionally, to the right of your title and description, select settings next to Reviewers, Assignees, Labels, Projects, or Milestone to add any of these options to your pull request.

    Note: These options offer different ways to collaborate using pull requests. For more information, see ‚ÄúAbout pull requests.‚Äù

    1. Select Create pull request.

    Your collaborators can now review your edits and make suggestions.

    Merge your pull request

    1. To merge the changes into main, go to the bottom of the pull request and select Merge pull request.

    2. Select Confirm merge.

    You will receive a message that the request was successfully merged and the request was closed.

    1. Select Delete branch.

    Now that your pull request is merged and your changes are on main, you can safely delete the readme-edits branch. If you want to make more changes to your project, you can always create a new branch and repeat this process.
