GitHub Basics Overview

Overview

The GitHub Basics tutorial provided is a revision of the “Hello World” exercise available on GitHub’s website. The purpose of this revision is to provide users with a simple, easy-to-use tutorial that has consistent formatting and terminology.

Document format

The "Hello World" exercise is referred to as both a quickstart guide and a tutorial, but these are different document types.

The following table provides a comparison of the two documents.

Document Audience Content
Tutorial Domain newbies More information
Quickstart guide Domain experts Less infomation

Document revisions

The following table provides a summary of the issues in the “Hello World” exercise and the solutions.

Issue Solution
Hello World is referred to as a quickstart guide and tutorial. Select a document type to create and use consistent terminology.
Conceptual information is included in the introduction. Include conceptual information about a feature in a separate section.
Prerequisites are in the introduction. To prepare users before they start, include a dedicated prerequisites section.
Conceptual information is included with the steps. Use the task section to instruct the user and include additional, informative content after the step.
The -ing verb form is used in a couple of headings. To prevent translation issues, don’t use the -ing verb form.
An input-specific verb, click, is used to describe interactions with user interface. Users interact with products using different input methods, so use generic verbs that work with any input method. Refer to an industry-relevant style guide for information about describing interactions with UI.
A summary of the skills learned are included in Next steps section. Summarize the skills and knowledge your users gained in a separate section.
